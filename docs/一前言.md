# 一、前言

这篇文章主要介绍两个工作流，一个是 **Flux2 Klein的高清修复工作流（单图 &批次）** ，一个是 **香蕉pro的合集工作流（高清放大 &高清修复）。**

**上一篇文章介绍过，高清放大其实对应了两个不同的场景。一个是“高清”，也就是把“模糊图片变得清晰”。另一个是放大，也就是把“小尺寸图片变成大尺寸图片（无损）”。**

**这两个不同的需求，对应了不同的技术路线。****其中“放大”，对应的是传统放大模型。**** 目前最好的放大模型就是SeedVR2.5。关于“高清放大”的原理分析，以及SeedVR2.5的工作流合集，可以看上一篇文章。**

**这篇文章主要介绍一下另一种需求场景“高清”，也就是如何把“模糊图片变得清晰”，以及对应的工作流。**

先看一下工作流的具体效果。

# 二、原理分析

想要实现“高清”的效果，有两种路线可以选择。

**第一种，利用“文生图大模型+超低重绘幅度”。**

**第二种，利用“编辑模型+提示词”。**

这里我选择的是第二种，利用Flux2 Klein的编辑能力，加上一些提示词来获得“高清”的效果。除了Klein之外，其他编辑模型均可，比如Flux Kontext，Qwen2511，以及香蕉pro。

**之所以选择Flux2 Klein，主要是Flux2 Klein比较轻便，对设备压力没那么大，效果也还算不错。高清放大算是一个刚需，部分朋友可能会有批量处理图片的需求。如果选择Qwen2511，对设备压力可能比较大。**

## 工作流相关介绍

### 基础介绍

Flux2 Klein的高清修复工作流一共有两版。一个是单图版，一个是批次版。两个版本的工作流可在线上运行，也可下载到本地部署。

本地版（夸克网盘）我用的是Flux2 Klein fp8量化模型，几乎对显存没有什么压力，8G以上都可以运行。如果显存在16G以上，可以选择9b模型，效果会更好（12G也可以试试，应该跑得通）。

线上版（RH）用的是9b模型，修复效果会更好一些，推荐使用线上版。

### 注意事项

工作流中还搭载了两个lora。一个是“真实感加强”的lora，这个一定要开。**还有一个是“增强一致性”的lora，这个可以选择关闭（快捷键：ctrl+B）。**

**这个lora端口，主要是为了后面解决Flux2 Klein的偏色问题而预留的一个端口。对当前版本的工作流，没有太大影响（开不开效果都差不多）。**

因为Flux2 Klein在对图像进行编辑的时候，有概率会出现这个问题。我在工作流中增加了一个“调色”节点，但只能缓解，并不能彻底解决这个问题。

**关于“偏色问题”，我咨询了一下社区的大佬，目前有一个解决方案就是使用小志大佬的9b lora。不过这个lora目前还在调试阶段，还没有发布，所以在这里预留了一个lora加载端口。**

我个人觉得，这个工作流的“偏色程度”可以忽略。**但如果对“偏色问题”比较敏感，想要一个“无偏色”的效果图，可以使用我的“香蕉****pro**** 高清修复工作流”。**

这个工作流需要调用香蕉pro的API，因为我选择的是RH的香蕉pro API，所以只能在runninghub上运行，无法在本地运行。如果想要在本地运行，可以把“API调用节点”换成Comfyui官方的调用节点，不过会贵一些。

把工作流中的“RH NanoBanana Pro gemini-3-pro-image-preview”节点，换成本地的“Nano Banana Pro（Google Gemini 图像）”节点。

充值后即可正常使用。

runninghub上调用香蕉pro API，1K图片前5张免费，后面的图片每张大概在3毛左右。从Comfyui的官方调用香蕉pro API，每张大概在7毛左右（我记不太清了）。

个人推荐，在runninghub上使用，经济实惠，效果几乎一样。

除此之外，香蕉pro工作流中，有两套提示词模板。或者说，包含了两个工作流。

**一个是用于“高清放大（跟SeedVR2.5效果差不多，无损放大）”，一个是用于“高清修复”。想要使用哪个工作流，可以通过“编号切换”节点来进行切换。**

**0：高清放大**

**1：高清修复**

除此之外，Flux2 Klein高清修复工作流中，还有一个红色节点“提示词开关”。

这个节点的作用是用来切换提示词，从而对应不同的场景（人物/场景）。在当前这个版本的工作流中，这个节点不用动，即便是处理一些“非人物图片”，也可以选择默认的“true”。

这个节点主要是为了后续迭代升级，预留的一个端口，跟前面的第二个lora加载器一样，当前版本可以暂时忽略。

# 五、使用方法

## 本地部署

**第一步，上传待处理图片/填写待处理图片文件夹地址。**

如果是处理“单图”的工作流，直接上传待处理图片即可。

如果是处理“批次”的工作流，在“加载图像列表(路径)”节点中的“路径”填写“待处理图片的文件夹路径”。同时在“图层工具：保存图像 Plus（高级）”的“自定义路径”中填写“处理图片的保存文件夹路径（可与上面一样）”

**第二步，选择本地模型** （RH上运行可忽略这步）。相关模型可在我的网盘下载。

**第三步，bypass掉第二个lora加载器（忘记也没事儿）**

**第四步，直接点击运行。**

##  线上运行

如果是在runninghub上运行这个工作流，单图操作完全一致，只需要上传待处理图片即可。但如果是批次操作，需要对工作流进行一下改动。

**由于runninghub无法识别电脑的本地路径，所以使用工作流中的“加载图像列表(路径)”节点，会出现无法识别“本地路径”的报错。**

个人推荐，把工作流下载到本地运行。如果想要在线上运行，需要把“加载图像列表(路径)”和“图层工具：保存图像 Plus（高级）”替换一下，具体效果如下图所示：

具体的节点和方法都有写，大家可以自行修改一下，难度不高。

香蕉pro的高清修复工作流，也是直接上传图片，点击运行即可。

如果想要无水印图片，可右键点击旁边的预览节点，选择“保存图像”，即可获得无水印图片。

不过这个工作流是调用香蕉pro的API，所以需要你的runninghub账户有余额，否则可能会失败。
